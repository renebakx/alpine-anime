Alpine.js meets AnimeJS

A tiny scratch-my-own-itch plugin for adding AnimeJS easings and viewport reveal presets through x-anime. It is built with AI help and the "You can just do things" mantra, not as a full AnimeJS API wrapper.

Scroll-Synced Parallax

x-anime.parallax links Anime.js WAAPI movement to scroll progress. The image layers below use local generated demo assets.

Fade Once + Slide Left

Use x-anime.fade.once for fade-in without movement.

Use x-anime.fade-left.once to slide in from the right and stay visible.

Slide Right + Fade Once

The image block starts from the left and settles in place only once.

The text block uses a pure one-time fade for an accessible, lower-motion pairing.

Fade In + Fade Out

This example uses x-anime.fade-in-out to fade in when entering the viewport and fade out when leaving it.

It intentionally does not use once, so the enter/leave behavior repeats while scrolling. This demo also starts earlier using enter.25 and exits sooner using leave.-10.

Bezier In + Bezier Out

The image uses x-anime.fade-up.ease.bezier-in for a deliberate start.

The copy uses x-anime.fade-up.ease.bezier-out for a softer finish.

Power Easing

ease.power-out.250 maps to Anime.js out(2.5).

ease.power-in.168 maps to Anime.js in(1.68).

Custom Preset

The demo registers blur-up before Alpine starts.

Use it like a built-in preset with x-anime.blur-up.once.

Fade Down + Threshold

x-anime.fade-down slides in from above. Pair it with threshold.70 to hold the trigger until 70% of the element is visible.

Threshold accepts whole percentages from 0 (any pixel visible) to 100 (fully in view). The default is 0.

Repeat

Add repeat to re-run the animation every time the element scrolls back into view. This is actually the default — once is the opt-out.
